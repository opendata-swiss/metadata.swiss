# Status forecast for end of June 2026

The first section gives a summary of the features implemented in the new ODS platform by end of June 2026.

The second section gives a summary of deferred features/requirements. Things that are not implemented in the new platform.


## Implemented in the platform by end of June 2026

### Harvester and metadata repository

The platform supports DCAT-AP CH V3 harvesting, for all harvesters in current ODS. Including harvesting from Geo-Portal.
A filter module discards datasets with missing or invalid license during harvesting.
For data publishers and the operator, the dashboard of Piveaus scheduler gives rudimentary and high-level monitoring of harvesting runs and shows the harvesting schedule.
Grafana provides more detailed and more technical insight into the harvesting for the operator (logs).

Custom controlled vocabularies (like ODS licenses and I14Y legalForms), catalog definitions, harvester configurations and harvester triggers (for scheduling harvesting runs) are all configured in files and kept under version control in git repository. Github actions (workflows) are used to interact with piveau on this operational level, for example to create or update a catalog, to update the harvesting schedule or to run a specific harvester on-demand outside of its automated schedule.

A "meta_harvester" is part of the system and used to create catalog definitions and harvester configurations based on current ODS content (using the CKAN API). When new publishers are onboarded on the current (CKAN) ODS, an operator can run the "meta_harvester" to replicate the new catalog and new harvester onto the new system (during testing phase until cut-over).

Organizations are managed in the same way as catalogs and harvesters, they are also kept under version control in git repository and updated in piveau using Github actions. Using the "meta_harvester", organizations are sourced from the I14Y API.


### Portal - Metadata

Dataset search and display. Distribution details and previews.

Organization search and display. Exploring datasets starting out from organization (as in the current ODS portal).


### Portal - Showcase, Handbook, Blog and static content

Showcase search and display. Showcase submission form. Showcase editing and workflow in CMS.

Handbook search and display. Handbook editing and workflow in CMS.

Blog search and display. Blog editing and workflow in CMS.

Static content display. Content editing and workflow in CMS.


### Portal - communication and personalization features

User can subscribe to Datasets and Themes and gets notifications by email.

User can comment on Datasets, Showcases, Handbook and Blogs.


### Portal - other

Statistics of portal usage via I14Y matomo integration.

## Deferred features/requirements

### Harvester and metadata repository

Managing datasets other than via harvesting (or API). Like manual editing or manual metadata upload, including Dataset workflow.

Multi tenant support in Piveau ("Mandantenfähigkeit").

Harvester WebUI for datapublisher (Config, test and schedule harvesters).

Extended error management and failure reporting for harvesting.

Showcase linked to Organization.

Showcase harvesting from other Swiss OGD portals.

Dataset machine translations.

Linkchecker on Distributions.

DCAT-AP CH conformance check.


### Portal - Metadata

Exposing DataService and DatasetSeries in the portal.

Dataset semantic search.

Metadata Quality dashboard.


### Portal - Showcase, Handbook, Blog and static content

Multimedia content in the Handbook. Running scripts inside the Handbook.

Keycloak Integration for CMS. Multi tenant support in CMS ("Mandantenfähigkeit"). Blog submission by external people.


### Portal - communication and personalization features

User can rate datasets and showcases ("5-stars").

Favorite Datasets and Showcases.

Onboardings ("Hilfefunktionen").

Message communication between datapublisher and operator.

Subscribing to Organizations and getting notifications by email.
